
## 1. 개요

**MoonBBang Station** — A space station introducing Munjeong's bakeries to aliens across the universe.

문정동 지역 베이커리를 분위기·목적·가격대 조건으로 필터링하고, 리뷰 기반 특징 분석을 통해 TOP 3 베이커리를 추천하는 웹 서비스.

지구인이든, 안드로메다에서 왔든, "지금 여기서 가장 맛있는 빵집이 어디야?"라는 질문에 3초 안에 답해주는 시스템이다. 아직 만난 적 없는 친구에게 보내는 빵집 가이드.

---

## 2. 문제 정의

### 2.1 지구인의 문제

문정동에는 베이커리가 정말 많이 있다. 지구인은 매일 점심시간에 "오늘은 어디 가지?"를 고민하며 평균 7분을 낭비한다. 이 7분이면 크루아상 하나를 먹을 수 있다.

### 2.2 외계인의 문제

지구에 처음 왔다. 빵이 뭔지 모른다. 가게가 뭔지 모른다. "아늑한 분위기"가 자신에게 적합한 환경인지 알 수 없다. 하지만 걱정할 필요 없다 — **이 시스템은 몇 가지 질문에 답하면, 최적의 빵 체험 장소를 알려준다.** 모르는 게 당연하고, 모르는 채로 와도 괜찮다.

### 2.3 시간여행자의 문제

2087년에서 왔는데 문정동 빵집 지도가 바뀌었다. 과거의 데이터라도 가격대와 분위기 필터로 빠르게 탐색할 수 있다.

## 4. User Story

### US-1. 베이커리 추천

- 사용자는 분위기(아늑한/모던한/감성적인/편안한), 목적(브런치/선물/모임/케이크/빵구경/식사빵/동네 단골/대형빵집), 가격대(일반/프리미엄) 중 원하는 조건을 선택할 수 있다.
- 사용자는 주차 가능, 주문제작 가능, 최대 거리 조건을 추가로 설정할 수 있다.
- 사용자는 조건을 선택하지 않아도 평점 기준 TOP 3 베이커리를 추천받을 수 있다.
- 사용자는 추천 결과에서 베이커리 이름, 주소, 대표 메뉴, 맛 프로필, 평점, 가격대, 거리, 주차/주문제작 정보, 특징 태그를 확인할 수 있다.
- 사용자는 추천 결과를 카카오맵 지도에서 마커로 확인할 수 있다.
- 사용자는 결과 페이지에서 "다시 추천받기"를 눌러 조건을 변경할 수 있다.

### US-2. 베이커리 목록 조회

- 사용자는 전체 베이커리 목록을 조회할 수 있다.
- 사용자는 특정 베이커리의 상세 정보를 조회할 수 있다.

### US-3. 리뷰 기반 특징 분석

- 시스템은 베이커리 리뷰 텍스트에서 키워드를 추출하여 특징 태그(예: 빵맛집, 케이크맛집, 선물하기좋은)를 자동 생성한다.
- 사용자는 추천 결과에서 리뷰 기반 특징 태그를 확인하여 베이커리 선택에 참고할 수 있다.

### US-4. 외계인 온보딩 — "처음이라 어려우시죠? 하나씩 알려드릴게요"

- 지구 음식 경험이 없는 사용자는 "빵이 뭔가요?" 도움말 버튼을 눌러 기초 설명을 볼 수 있다.
- 시스템은 각 분위기 옵션에 감각적 설명을 제공한다 (예: "아늑한 = 체온이 0.3도 올라가는 느낌의 공간").
- 시스템은 각 대표 메뉴에 재료·식감·맛 프로필 설명을 제공한다 (예: "크루아상 = 겉: 바삭, 속: 부드러움, 버터 향, 한 입이면 지구가 좋아진다").
- 사용자가 처음 방문하면, "어서 와, 우주에서 온 친구" 환영 메시지를 볼 수 있다.
- 어떤 단계에서든 "잘 모르겠어요"를 선택할 수 있고, 시스템이 대신 골라준다.

### US-5. 감각 기반 추천 — "말로 설명하기 어렵다면, 느낌으로 찾아보세요"

- 지구의 오감 체계에 익숙하지 않은 사용자를 위해, 감각 기반 질문으로 추천할 수 있다.
  - "바삭한 식감 vs 부드러운 식감?" → 빵 종류 필터
  - "달콤한 맛 vs 고소한 맛?" → 메뉴 카테고리 필터
  - "혼자 조용히 vs 여럿이 왁자지껄?" → 분위기 필터

### US-6. 재방문 유도 — "문정동은 항상 여기서 기다릴게요"

- 추천 결과 페이지 하단에 항상 따뜻한 재방문 초대 메시지가 표시된다.
- "또 놀러 오세요", "문정동은 항상 여기 있어요", "다음엔 다른 빵도 먹어봐요" 등 랜덤 메시지.
- 결과 페이지에서 "이 빵집 기억해두기" 기능으로 다음 방문 시 참고할 수 있다.

## 5. 기능 요구사항

### FR-1. 베이커리 추천

| ID | 요구사항 |
|----|---------|
| FR-1.1 | 분위기, 목적, 가격대 조건을 조합하여 추천 요청할 수 있어야 한다 |
| FR-1.2 | 각 조건은 선택 사항이며, 미입력 시 해당 조건을 무시한다 |
| FR-1.3 | 주차 가능, 주문제작 가능은 정확 일치 필터로 동작한다 |
| FR-1.4 | 최대 거리 설정 시 문정역 기준 반경 내 베이커리만 필터링한다 |
| FR-1.5 | 점수 계산: rating × 0.5 + mood 일치 +2 + purpose 일치 +2 + price_range 일치 +1 + 거리 보너스 × 3 + 랜덤 다양화 0~1.5 (상세는 Tech Spec 참고) |
| FR-1.6 | 점수 내림차순 정렬 후 상위 3개를 반환한다 |
| FR-1.7 | 조건에 매칭되는 베이커리가 없어도 rating 순으로 TOP 5을 반환한다 (빈 결과 없음) |

| FR-2.2 | GET `/api/bakeries/{id}` — 단일 베이커리를 JSON으로 반환한다 |
| FR-2.3 | 존재하지 않는 id 조회 시 404 응답을 반환한다 |

| ID | 요구사항 |
|----|---------|

| FR-3.2 | 키워드-태그 매핑: 아늑한, 빵맛집, 케이크맛집, 마카롱맛집, 가성비좋은, 줄서는집, 선물하기좋은, 브런치맛집, 식사빵, 동네 단골, 대형빵집 |
